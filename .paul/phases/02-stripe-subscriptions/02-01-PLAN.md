  <name>Task 1: Install stripe and create migration 007</name>
  <files>package.json, supabase/migrations/007_stripe_subscriptions.sql</files>
  <action>
    1. Run `npm install stripe` to add the stripe package.

    2. Create migration file `supabase/migrations/007_stripe_subscriptions.sql`:

    ```sql
    -- Add tier column to farms
    ALTER TABLE farms ADD COLUMN IF NOT EXISTS tier TEXT NOT NULL DEFAULT 'free';

    -- Subscriptions table
    CREATE TABLE IF NOT EXISTS subscriptions (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
      farm_slug TEXT NOT NULL,
      stripe_customer_id TEXT,
      stripe_subscription_id TEXT UNIQUE,
      stripe_price_id TEXT,
      tier TEXT NOT NULL DEFAULT 'free',
      status TEXT NOT NULL DEFAULT 'active',
      current_period_end TIMESTAMPTZ,
      created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL
    );

    -- Indexes
    CREATE INDEX IF NOT EXISTS subscriptions_user_id_idx ON subscriptions(user_id);
    CREATE INDEX IF NOT EXISTS subscriptions_farm_slug_idx ON subscriptions(farm_slug);
    CREATE UNIQUE INDEX IF NOT EXISTS subscriptions_stripe_subscription_id_idx
      ON subscriptions(stripe_subscription_id) WHERE stripe_subscription_id IS NOT NULL;

    -- RLS
    ALTER TABLE subscriptions ENABLE ROW LEVEL SECURITY;

    DO $$ BEGIN
      CREATE POLICY "owner_read_subscriptions" ON subscriptions
        FOR SELECT USING (auth.uid() = user_id);
    EXCEPTION WHEN duplicate_object THEN NULL; END $$;

    DO $$ BEGIN
      CREATE POLICY "service_role_all_subscriptions" ON subscriptions
        FOR ALL USING (auth.role() = 'service_role');
    EXCEPTION WHEN duplicate_object THEN NULL; END $$;
    ```

    Apply this migration manually in Supabase SQL Editor:
    https://supabase.com/dashboard/project/eqrmwkyzllkpkuqhwswk/sql

  </action>

<task type="auto">
  <name>Task 2: POST /api/stripe/checkout — create checkout session</name>
  <files>src/app/api/stripe/checkout/route.ts</files>
  <action>
    Create `src/app/api/stripe/checkout/route.ts`:

    - Method: POST only
    - Auth: require Bearer token from Authorization header — validate against Supabase to get user. Return 401 if absent/invalid.
    - Body schema (Zod): `{ farm_slug: z.string().min(1), price_id: z.string().startsWith('price_') }`
    - If STRIPE_SECRET_KEY is not set, return 503 `{ error: 'Platby nejsou momentálně dostupné.' }`
    - If NEXT_PUBLIC_SITE_URL is not set, default base URL to 'https://mapafarem.cz'
    - Create Stripe Checkout Session:
      ```
      mode: 'subscription'
      payment_method_types: ['card']
      line_items: [{ price: price_id, quantity: 1 }]
      metadata: { farm_slug, user_id: user.id }
      success_url: `${baseUrl}/pro-farmary?upgrade=success`
      cancel_url: `${baseUrl}/pro-farmary#pricing`
      ```
    - Return `{ url: session.url }` with status 200
    - On Stripe error, return 500 with `{ error: 'Nepodařilo se vytvořit platbu.' }`

    Use `import Stripe from 'stripe'` and instantiate with:
    ```ts
    const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, {
      apiVersion: '2024-12-18.acacia'
    })
    ```
    Use `getSupabaseClient()` from `@/lib/supabase` to verify the Bearer token via `supabase.auth.getUser(token)`.
    Do NOT use `getSupabaseRaw()` here — we're only reading auth, not writing to untyped tables.

  </action>
  <verify>`npm run build` passes with no TypeScript errors on this file.</verify>
  <done>AC-3 satisfied: endpoint creates Stripe Checkout Session when STRIPE_SECRET_KEY present, returns 503 when absent.</done>
</task>

<task type="auto">
  <name>Task 3: POST /api/stripe/webhook — handle subscription lifecycle</name>
  <files>src/app/api/stripe/webhook/route.ts</files>
  <action>
    Create `src/app/api/stripe/webhook/route.ts`:

    - Export `export const dynamic = 'force-dynamic'` and `export const runtime = 'nodejs'` (webhooks need raw body)
    - Read raw body: `const body = await req.text()`
    - Verify signature using `stripe.webhooks.constructEvent(body, sig, process.env.STRIPE_WEBHOOK_SECRET!)`
    - On invalid signature, return 400 `{ error: 'Invalid signature' }`
    - If STRIPE_SECRET_KEY or STRIPE_WEBHOOK_SECRET absent, return 503
    - Use `getSupabaseRaw()` from `@/lib/supabase` for all DB writes (subscriptions table not in Database type)

    Handle these events:

    **checkout.session.completed:**
    - Extract `farm_slug`, `user_id` from `session.metadata`
    - Extract `subscription_id = session.subscription` (string)
    - Retrieve the subscription from Stripe: `stripe.subscriptions.retrieve(subscription_id)`
    - Map `price_id` → tier: check against env vars `STRIPE_PRICE_PROFESIONAL` and `STRIPE_PRICE_PREMIUM`
      - If price matches STRIPE_PRICE_PROFESIONAL → tier = 'profesional'
      - If price matches STRIPE_PRICE_PREMIUM → tier = 'premium'
      - Default → tier = 'profesional' (safer fallback than 'free')
    - Upsert into `subscriptions` (on conflict stripe_subscription_id):
      `{ user_id, farm_slug, stripe_customer_id, stripe_subscription_id, stripe_price_id, tier, status: 'active', current_period_end }`
    - Update `farms` set `tier = tier` where `slug = farm_slug`

    **customer.subscription.updated:**
    - Extract subscription object from event.data.object
    - Update subscriptions row: status, current_period_end, tier (re-derive from price)
    - Update farms.tier accordingly

    **customer.subscription.deleted:**
    - Find subscription row by stripe_subscription_id
    - Set status = 'canceled'
    - Set farms.tier = 'free' where slug = farm_slug

    Always return 200 `{ received: true }` after processing (Stripe requires 200 to stop retrying).
    Log errors to console.error but do not throw — always return 200 unless signature is invalid.

    Avoid: `next/headers` in this route — use `req.headers.get()` directly.

  </action>
  <verify>`npm run build` passes. Route file exists at correct path with no TypeScript errors.</verify>
  <done>AC-4 satisfied: webhook handler processes all three subscription lifecycle events, updates DB, resets tier on cancellation.</done>
